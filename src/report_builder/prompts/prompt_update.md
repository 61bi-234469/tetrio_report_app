# TETR.IOレポート差分更新プロンプト

既存の分析定義・HTML雛形を変更せず、新しい入力データによって変化した部分だけを更新してください。

## 入力
- 新しいCSVまたはJSON
- `cache/manifest.json`
- 必要に応じて `cache/ai_payload.json`
- 更新対象の章HTMLだけ
- 変更が必要なグラフのPNGだけ

## 変更禁止
- 指標定義と勝敗判定
- JST変換
- セッション定義（前試合完了直後から次試合開始まで10分以内なら同一セッション）
- タイブレーク定義
- 能力レーダーの軸順
- 4スタイルの配置
- CSS、章構成、HTMLクラス名
- 変更対象外の章・グラフ・付録

## 手順
1. 入力データのSHA-256をmanifestと比較する。
2. 同一なら再集計せず、既存キャッシュを使う。
3. 異なる場合も、前回データとの差分と影響する集計だけを更新する。
4. 更新された事実を `analysis_summary.json` に保存する。
5. 変更があるグラフだけPNGを再生成する。
6. 説明文は「読み方・結果・解釈・注意・要点」の該当部分だけ更新する。
7. 出力は更新対象ファイルだけとし、完成HTML全文を会話へ貼らない。
8. 最後にローカルの `scripts/build_report.py` で自己完結HTMLを生成する。

## 粒度
- 各図表の粒度（試合単位・ラウンド単位・セッション内の試合位置）は必ず残す。
- 粒度が混在する章では、どの文がどの単位を扱うかを本文で明示する。
- 「セッション勝率」「ラウンド単位の試合勝率」のように粒度が混ざる表現を作らない。

## 品質条件
- 数値は実データから再計算する。
- 旧値を無検証で引き継がない。
- 変更のない内容は書き直さない。
- Base64画像をAIへの入力・出力に含めない。
